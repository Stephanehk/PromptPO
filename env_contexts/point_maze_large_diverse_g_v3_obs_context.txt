# Observation — env_name point_maze_large_diverse_g_v3 (Gymnasium-Robotics PointMaze_Large_Diverse_G-v3 with explicit maze_map)
# Generic Point Maze mechanics: https://robotics.farama.org/envs/maze/point_maze/

# Policies and Feedback receive a `PointMazeLargeDiverseGV3Observation` instance at each step
# (class in direct_policy_learning.observations.point_maze_large_diverse_g_v3_observation).
#
# The underlying Gymnasium environment returns a dict observation with three keys.
# The wrapper copies them into these attributes (float64 numpy arrays):
#
# (1) observation_vector — shape (4,)
#     [0] ball x position in world frame (m)
#     [1] ball y position in world frame (m)
#     [2] ball x velocity (m/s)
#     [3] ball y velocity (m/s)
#
# (2) desired_goal — shape (2,)
#     [0] target / goal x (m) — red target site in the maze (one of the g cells for this episode)
#     [1] target / goal y (m)
#
# (3) achieved_goal — shape (2,)
#     [0] current ball x (m) — same as observation_vector[0]
#     [1] current ball y (m) — same as observation_vector[1]
#
# At reset, a goal cell is chosen from cells marked g and a reset cell from cells marked r,
# then uniform noise on the cell center is applied.
